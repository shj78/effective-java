
# Effective Java

Day 4

## Item 6 : Avoid creating unnecessary objects

1. 똑같은 기능의 객체는 생성 말고 재사용하자
   1. 재사용이 필요한 경우
      1. 불변객체일 때
         - new String("Deep sea") 
           - 자바 9부터 deprecated API로 지정됐다
           - 그러니 팩터리 메서드를 쓰자
         - cf) 가변인데도 재사용하는 경우도 있다. 
         - Ex) Map 인터페이스의 keySet
      2. 캐싱으로 객체 생성 비용을 줄일 수 있을 때
         - 주의! 캐싱같이 따로 초기화한 후에 한번도 호출하지 않는다면 쓸 데 없이 초기화 된 것이다.
         - 호출될 때 필드를 초기화하는 지연 초기화(Lazy)방식도 있지만, 코드가 복잡하고 성능적으로 개선이 없다.
   2. 불필요한 객체를 만들어내는 경우
      1. 오토 박싱
2. 커스텀 객체풀은 특정한 상황이 아니고서야 지양하자
   - GC가 워낙 잘 되어있다
   - Ex) 데이터베이스 객체 풀은 커스텀 객체풀 인정한다
3. 더 알아볼 것
   1. Deprecated
   2. 정규 표현식
   3. GC
