### Item 12 : Always override toString()

### Enum, 정적 유틸 클래스를 제외하고 toString은 재정의 하는 게 좋다
> toString은 디버거, print, assert문, 오류 로깅등에서 호출된다. 이렇게 자주 볼 수 있는 toString()이라면 개발자에게 도움이 되는 형태로 출력되어야 한다.
#### 왜 그런가?
1. toString()은 클래스이름@16진수 변환된 hashCode값만으로 출력된다. 개발자는 이를 보고 어떤 정보도 얻을 수 없다.
2. 해당 클래스의 구체적인 정보를 개발자에게 전달하기 위해서, 이는 자바 API 노트에도 권장되어있다.
3. 디버깅이 쉬워진다.
#### 어떻게 재정의 하는 게 좋은가?
1. 객체의 주요 정보를 정의한다.
2. 주요 정보가 너무 많다면 요약해서 재정의 한다.
3. API DOC으로 정리될 수 있게끔 주석을 달자.
4. 주석이 어렵다면 의도라도 적자.
- 올바른 toString 재정의의 예
```java 
    /**
     * 이 전화번호의 문자열 표현을 반환한다.
     * 이 문자열은 "XXX-YYY-ZZZZ" 형태의 12글자로 구성된다.
     * XXX는 지역 코드, YYY는 프리픽스, ZZZZ는 가입자 번호다.
     * 각각의 대문자는 10진수 숫자 하나를 나타낸다.
     *
     * 전화번호의 각 부분의 값이 너무 작아서 자릿수를 채울 수 없다면,
     * 앞에서부터 0으로 채워나간다. 예컨대 가입자 번호가 123이라면
     * 전화번호의 마지막 네 문자는 "0123"이 된다.
     */
    @Override public String toString() {
        return String.format("%03d-%03d-%04d",
                areaCode, prefix, lineNum);
    }

```
